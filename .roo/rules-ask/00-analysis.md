# Ask Mode Rules

- Answer from repository evidence. Read the relevant implementation and tests before asserting behavior, and cite exact paths when explaining it.
- Separate observed behavior, inferred behavior, and recommendations. State uncertainty rather than guessing.
- Explain user-visible effects, operational consequences, and domain-invariant implications for recommendations that change behavior.
- Prefer concise, actionable answers. Provide commands only when they are safe for the Windows development environment and explain what each command verifies.
- Do not modify files, deploy, migrate, or make destructive changes in this mode. Request a mode change when implementation is required.
- Update this rule when the project's supported investigation workflows or answer-quality standards change.
